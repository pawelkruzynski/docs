## Objective

OVHcloud provides permission groups that group together all of the actions required for specific use cases.

This guide lists all of these permission groups and details the actions contained in each group.

## Requirements

To use permission groups, you will need the following:

- You have an [OVHcloud customer account](/pages/account_and_service_management/account_information/ovhcloud-account-creation)
- You know [how to set up IAM policies via the Control Panel](/pages/account_and_service_management/account_information/iam-policy-ui)
- You know [how to implement IAM policies via the API](/pages/account_and_service_management/account_information/iam-policies-api)

## Instructions

Permission groups allow you to set up turn-key access policies without having to know the full list of necessary actions.
This list will change over time as new permission groups are created.

If a permission group is updated, a new permission group will be created and the old permission group will be deprecated to avoid impacting existing IAM policies.

### Using permission groups

#### Listing permission groups

The permission groups are accessible via the following API:

|**Method**|**Path**|**Description**|
| :-: | :-: | :-: |
|GET|/iam/permissionsGroup|Retrieve all permission groups|

```json
{
    "id": "00000000-0000-0000-0001-000000000001",
    "urn": "urn:v1:eu:permissionsGroup:ovh:globalAdmin",
    "name": "globalAdmin",
    "owner": "ovh",
    "description": "Give global admin access across all OVHcloud products",
    "permissions": {
        "allow": [
        {
            "action": "*"
        }
        ]
    },
    "createdAt": "2023-03-14T09:10:57.40418Z",
    "updatedAt": null
},
```

#### Use permission groups from the Control Panel

These permission groups can then be used in addition to or in place of unit actions in access policies:

![Permissions groups UI](images/permissions_groups_ui.png){.thumbnail}

#### Using permission groups in the API

These permission groups can then be used in addition to or in place of unit actions in access policies:

```json
{
  "description": "",
  "identities": [...],
  "name": "",
  "permissions": {
    "allow": [
      {
        "action": "..."
      },
    ]
  },
  "permissionsGroups": [
      {
        "urn": "urn:v1:eu:permissionsGroup:ovh:globalAdmin"
      }
  ],
  "resources": [...]
}
```
